## 📖 ພາກນຳ

### ກ່ຽວກັບລະບົບ LIS

**LIS (Laboratory Information System)** ແມ່ນລະບົບບໍລິຫານຈັດການຂໍ້ມູນຫ້ອງທົດລວດ ທີ່ຊ່ວຍໃນການ:
- ✅ ຈັດການໃບສັ່ງກວດທາງຫ້ອງທົດລວດ
- ✅ ຕິດຕາມສະຕັອກນ້ຳຢາ ແລະ ເຄື່ອງໃຊ້
- ✅ ວິເຄາະຂໍ້ມູນສະຖິຕິຕ່າງໆ
- ✅ ບຳລຸງຮັກສາເຄື່ອງຈັກ ແລະ ອຸປະກອນ
- ✅ ຈັດການຜົນກວດ ແລະ ພິມລາຍງານ

### ຄຸນສົມບັດເດັ່ນ

| ຄຸນສົມບັດ | ຄຳອະທິບາຍ |
|------------|------------|
| 🚀 ວ່ອງໄວ | ໃຊ້ເຕັກໂນໂລຊີທັນສະໄໝ ໂຫຼດໄວ |
| 📱 Responsive | ໃຊ້ງານໄດ້ທັງຄອມພິວເຕີ ແລະ ມືຖື |
| 🔒 ປອດໄພ | ມີລະບົບ Login ແລະ ກຳນົດສິດ |
| 📊 Analytics | Dashboard ວິເຄາະຂໍ້ມູນແບບ Real-time |
| 📦 Inventory | ຈັດການສະຕັອກແບບ FIFO ພ້ອມແຈ້ງເຕືອນ |
| 🖨️ Print Ready | ພິມສະຕິກເກີ, ໃບຮັບເງິນ, ຜົນກວດ |

## 📖 ຂັ້ນຕອນການໃຊ້ງານແຕ່ລະໝວດ

### 1️⃣ 📊 DASHBOARD (ພາບລວມ)

#### ຈຸດປະສົງ
ສະແດງຂໍ້ມູນສະຖິຕິການໃຊ້ງານ ແລະ ກຣາຟວິເຄາະ

#### ວິທີໃຊ້ງານ

**ຂັ້ນຕອນທີ 1:** ເລືອກຊ່ວງວັນທີ
- ກົດປຸ່ມ: **ມື້ນີ້**, **ອາທິດນີ້**, **ເດືອນນີ້**, **ປີນີ້**
- ຫຼື ເລືອກວັນທີເອງ: **ຈາກ** → **ຫາ**

**ຂັ້ນຕອນທີ 2:** ໃສ່ຕົວກັ່ນຕອງ (ຖ້າຕ້ອງການ)
- **ພະແນກ**: ເລືອກພະແນກທີ່ຕ້ອງການເບິ່ງ
- **ແພດສົ່ງກວດ**: ເລືອກແພດສະເພາະ
- **ປະເພດການກວດ**: Normal ຫຼື Package
- **ໝວດໝູ່ການກວດ**: Biochemistry, Hematology, ຯລຯ

**ຂັ້ນຕອນທີ 3:** ກົດປຸ່ມ **🔍 ໂຫຼດຂໍ້ມູນ**

**ຂັ້ນຕອນທີ 4:** ຊົມຜົນວິເຄາະ

#### ຂໍ້ມູນທີ່ສະແດງ

##### KPI Cards (ຕົວເລກສຳຄັນ)
| ຕົວເລກ | ຄຳອະທິບາຍ |
|--------|------------|
| 👥 ຈຳນວນບິນ | ຈຳນວນໃບສັ່ງກວດທັງໝົດໃນຊ່ວງທີ່ເລືອກ |
| 💰 ລາຍຮັບລວມ | ຍອດເງິນລວມຈາກການກວດ |
| 🏥 IN-LAB | ລາຍຮັບຈາກການກວດໃນຫ້ອງທົດລວດ |
| 🚚 OUTSOURCE | ລາຍຮັບຈາກການສົ່ງກວດນອກ |

##### ກຣາຟສະແດງຜົນ
1. **ເພດຄົນເຈັບ** (Pie Chart) - ສັດສ່ວນ Male/Female
2. **OPD vs IPD** (Doughnut) - ຜູ້ປ່ວຍນອກ vs ຜູ້ປ່ວຍໃນ
3. **Visit Type** (Pie) - ປະເພດການມາຮັບບໍລິການ
4. **Normal vs Package** - ການກວດແບບດ່ຽວ vs ແບບຊຸດ
5. **In-Lab vs Outsource** - ກວດໃນ vs ສົ່ງນອກ
6. **ລາຍຮັບຕາມແພດ** (Bar) - ແພດແຕ່ລະຄົນສົ່ງກວດເທົ່າໃດ
7. **ລາຍຮັບຕາມພະແນກ** (Bar) - ພະແນກແຕ່ລະແຫ່ງສົ່ງກວດເທົ່າໃດ
8. **ກຸ່ມອາຍຸ** (Bar) - ຄົນເຈັບແຕ່ລະກຸ່ມອາຍຸ
9. **Time Slot** (Bar+Line) - ຈຳນວນບິນ ແລະ ລາຍຮັບຕາມຊ່ວງເວລາ

##### ຕາຕະລາງສະຫຼຸບ
- **Top 5 Tests**: 5 ລາຍການກວດທີ່ມີຄົນສັ່ງຫຼາຍທີ່ສຸດ
- **Top 5 Categories**: 5 ໝວດໝູ່ທີ່ມີຄົນສັ່ງຫຼາຍທີ່ສຸດ
- **Summary Table**: ຕາຕະລາງສະຫຼຸບລາຍການກວດທັງໝົດ

### 2️⃣ 📝 ORDER FORM (ແບບສັ່ງກວດ)

#### ຈຸດປະສົງ
ສ້າງໃບສັ່ງກວດໃໝ່ສຳລັບຄົນເຈັບ

#### ວິທີໃຊ້ງານ

##### ສ່ວນທີ 1: ຂໍ້ມູນຄົນເຈັບ

**ຂັ້ນຕອນທີ 1:** ປ້ອນ **Patient ID**
```
ຕົວຢ່າງ: HN-12345
```
- ລະບົບຈະຄົ້ນຫາຂໍ້ມູນຄົນເຈັບອັດຕະໂນມັດ
- ຖ້າພົບ: ຊື່, ອາຍຸ, ເພດ ຈະຖືກດຶງມາອັດຕະໂນມັດ
- ຖ້າບໍ່ພົບ: ສາມາດປ້ອນຂໍ້ມູນດ້ວຍມື

**ຂັ້ນຕອນທີ 2:** ປ້ອນຂໍ້ມູນຄົນເຈັບ
- **ຊື່ ແລະ ນາມສະກຸນ**: ປ້ອນຊື່ຄົນເຈັບ
- **ອາຍຸ**: ປ້ອນອາຍຸ (ເປັນປີ)
- **ເພດ**: ເລືອກ Male ຫຼື Female

**ຂັ້ນຕອນທີ 3:** ເລືອກຂໍ້ມູນການມາຮັບບໍລິການ
- **ວັນທີ & ເວລາ**: ວັນທີ ແລະ ເວລາມາຮັບບໍລິການ
- **Time Slot**: ເລືອກຊ່ວງເວລາ
  - `08:00-16:00` (ເຊົ້າ)
  - `16:00-21:00` (ແລງ)
  - `21:00-08:00` (ກະເດີກ)
- **Visit Type**: ປະເພດການມາຮັບບໍລິການ (OPD, IPD, Emergency)
- **Insite**: ສະຖານທີ່ມາຮັບບໍລິການ

**ຂັ້ນຕອນທີ 4:** ເລືອກບຸກຄະລາກອນ
- **Doctor**: ແພດຜູ້ສົ່ງກວດ
- **Department**: ພະແນກ
- **Sender**: ຜູ້ສົ່ງຕົວ

**ຂັ້ນຕອນທີ 5:** ເລືອກປະເພດການກວດ
- ☑️ **ສົ່ງກວດນອກ**: ຖ້າຕ້ອງສົ່ງກວດນອກ ໃຫ້ກາໝາຍ
- **Lab Destination**: ເລືອກຫ້ອງທົດລວດປາຍທາງ

##### ສ່ວນທີ 2: ເລືອກລາຍການກວດ

**ຂັ້ນຕອນທີ 1:** ເລືອກປະເພດການກວດ
- **Normal**: ກວດເປັນລາຍການດ່ຽວ
- **Package**: ກວດເປັນຊຸດ

**ຂັ້ນຕອນທີ 2:** ຄົ້ນຫາລາຍການກວດ
- ພິມຊື່ລາຍການໃນຊ່ອງ **ຄົ້ນຫາລາຍການ...**
- ຕົວຢ່າງ: "CBC", "Cholesterol", "Glucose"

**ຂັ້ນຕອນທີ 3:** ເລືອກລາຍການກວດ
- ☑️ ກາໝາຍໃສ່ລາຍການທີ່ຕ້ອງການ
- ລາຄາຈະຖືກຄິດໄລ່ອັດຕະໂນມັດ

**ຂັ້ນຕອນທີ 4:** ກວດສອບຕະກຣ້າ
- ລາຍການທີ່ເລືອກຈະສະແດງໃນຕະກຣ້າ
- ລາຄາລວມຈະສະແດງດ້ານລຸ່ມ

##### ສ່ວນທີ 3: ບັນທຶກໃບສັ່ງກວດ

**ຂັ້ນຕອນທີ 1:** ກວດສອບຂໍ້ມູນ
- ກວດສອບຄວາມຖືກຕ້ອງຂອງຂໍ້ມູນທັງໝົດ

**ຂັ້ນຕອນທີ 2:** ກົດປຸ່ມ **💾 ບັນທຶກໃບສັ່ງກວດ**

**ຂັ້ນຕອນທີ 3:** ລໍຖ້າລະບົບປະມວນຜົນ
- ລະບົບຈະສ້າງ **Order ID** ອັດຕະໂນມັດ
- ຕັດສະຕັອກນ້ຳຢາອັດຕະໂນມັດ
- ບັນທຶກລົງຖານຂໍ້ມູນ

**ຂັ້ນຕອນທີ 4:** ພິມສະຕິກເກີ
- ກົດປຸ່ມ **🖨️ ພິມສະຕິກເກີ**
- ຕິດສະຕິກເກີໃສ່ຕົວຢ່າງ

#### ການແກ້ໄຂໃບສັ່ງກວດ

1. ໄປທີ່ **ປະຫວັດສັ່ງກວດ**
2. ຄົ້ນຫາໃບສັ່ງກວດທີ່ຕ້ອງການ
3. ກົດປຸ່ມ **✏️ ແກ້ໄຂ**
4. ແກ້ໄຂຂໍ້ມູນ
5. ກົດ **💾 ບັນທຶກ**

#### ການຍົກເລີກໃບສັ່ງກວດ

1. ໄປທີ່ **ປະຫວັດສັ່ງກວດ**
2. ຄົ້ນຫາໃບສັ່ງກວດທີ່ຕ້ອງການ
3. ກົດປຸ່ມ **🗑️ ລຶບ**
4. ຢືນຢັນການລຶບ

### 3️⃣ 📋 HISTORY (ປະຫວັດສັ່ງກວດ)

#### ຈຸດປະສົງ
ກວດເບິ່ງ ແລະ ຄົ້ນຫາປະຫວັດໃບສັ່ງກວດ

#### ວິທີໃຊ້ງານ

**ຂັ້ນຕອນທີ 1:** ກຳນົດຊ່ວງວັນທີ
- **ຈາກ**: ວັນທີເລີ່ມຕົ້ນ
- **ຫາ**: ວັນທີສິ້ນສຸດ

**ຂັ້ນຕອນທີ 2:** ໃສ່ຕົວກັ່ນຕອງ (ຖ້າຕ້ອງການ)
- **Order ID**: ເລກທີໃບສັ່ງກວດ
- **Patient ID**: ເລກທີຄົນເຈັບ
- **Patient Name**: ຊື່ຄົນເຈັບ
- **Status**: ສະຖານະ (Pending, Completed, Cancelled)

**ຂັ້ນຕອນທີ 3:** ກົດປຸ່ມ **🔍 ຄົ້ນຫາ**

**ຂັ້ນຕອນທີ 4:** ຊົມຜົນການຄົ້ນຫາ

#### ການຈັດການໃບສັ່ງກວດ

| ການກະທຳ | ວິທີເຮັດ |
|----------|----------|
| 👁️ ເບິ່ງລາຍລະອຽດ | ກົດປຸ່ມ **👁️ ເບິ່ງ** |
| ✏️ ແກ້ໄຂ | ກົດປຸ່ມ **✏️ ແກ້ໄຂ** |
| 🖨️ ພິມ | ກົດປຸ່ມ **🖨️ ພິມ** |
| 🗑️ ລຶບ | ກົດປຸ່ມ **🗑️ ລຶບ** |

#### ການສົ່ງອອກ Excel

1. ກົດປຸ່ມ **📊 Export Excel**
2. ເລືອກຂໍ້ມູນທີ່ຕ້ອງການສົ່ງອອກ
3. ກົດ **Download**

### 4️⃣ 📦 INVENTORY (ສາງນ້ຳຢາ)

#### ຈຸດປະສົງ
ຈັດການສະຕັອກນ້ຳຢາ ແລະ ເຄື່ອງໃຊ້ໃນຫ້ອງທົດລວດ

#### ວິທີໃຊ້ງານ

##### ການເພີ່ມນ້ຳຢາໃໝ່ (Reagent Master)

**ຂັ້ນຕອນທີ 1:** ກົດປຸ່ມ **➕ ເພີ່ມນ້ຳຢາໃໝ່**

**ຂັ້ນຕອນທີ 2:** ປ້ອນຂໍ້ມູນ
- **ຊື່ນ້ຳຢາ**: ຊື່ Reagent
- **ຫົວໜ່ວຍ**: ຫົວໜ່ວຍນັບ (vial, test, ml, ຯລຯ)

**ຂັ້ນຕອນທີ 3:** ກົດ **💾 ບັນທຶກ**

##### ການຮັບເຂົ້າ Lot ໃໝ່

**ຂັ້ນຕອນທີ 1:** ກົດປຸ່ມ **📥 ຮັບເຂົ້າ Lot ໃໝ່**

**ຂັ້ນຕອນທີ 2:** ປ້ອນຂໍ້ມູນ Lot
- **ນ້ຳຢາ**: ເລືອກຊື່ນ້ຳຢາ
- **Lot No**: ເລກທີ Lot
- **ຜູ້ສະໜອງ**: ຊື່ບໍລິສັດຜູ້ສະໜອງ
- **ສະຖານທີ່ເກັບ**: ຕູ້ເຢັນ/ຊັ້ນວາງ
- **ວັນຮັບ**: ວັນທີຮັບເຂົ້າ
- **ວັນໝົດອາຍຸ**: ວັນທີສິ້ນສຸດອາຍຸ
- **ຈຳນວນ**: ຈຳນວນທີ່ຮັບເຂົ້າ

**ຂັ້ນຕອນທີ 3:** ກົດ **💾 ບັນທຶກ**

**ຂັ້ນຕອນທີ 4:** ລະບົບຈະ
- ສ້າງ **Lot ID** ອັດຕະໂນມັດ
- ບັນທຶກ **IN Transaction**
- ອັບເດດສະຕັອກລວມ

##### ການເບີກນ້ຳຢາອອກ

**ຂັ້ນຕອນທີ 1:** ກົດປຸ່ມ **📤 ເບີກນ້ຳຢາ**

**ຂັ້ນຕອນທີ 2:** ປ້ອນຂໍ້ມູນ
- **ນ້ຳຢາ**: ເລືອກຊື່ນ້ຳຢາ
- **ຈຳນວນ**: ຈຳນວນທີ່ເບີກ
- **ໝາຍເຫດ**: ເຫດຜົນການເບີກ

**ຂັ້ນຕອນທີ 3:** ກົດ **💾 ບັນທຶກ**

**ຂັ້ນຕອນທີ 4:** ລະບົບຈະ
- ຕັດສະຕັອກແບບ **FIFO** (ເຂົ້າກ່ອນ-ອອກກ່ອນ)
- ບັນທຶກ **OUT Transaction**

##### ການຕິດຕາມສະຖານະ

**ສະຖານະນ້ຳຢາ:**
| ສະຖານະ | ສີ | ຄຳອະທິບາຍ |
|--------|-----|------------|
| ✅ Normal | 🟢 | ປົກກະຕິ |
| ⚠️ Expiring Soon | 🟡 | ໃກ້ໝົດອາຍຸ (< 30 ມື້) |
| 🚫 Expired | 🔴 | ໝົດອາຍຸແລ້ວ |
| ❌ Out of Stock | ⚫ | ນ້ຳຢາໝົດສະຕັອກ |

**ການແຈ້ງເຕືອນ:**
- ລະບົບຈະສະແດງ **Badge ສີແດງ** ດ້ານເທິງເມື່ອມີນ້ຳຢາໃກ້ໝົດອາຍຸ
- ກົດທີ່ **🔔** ເພື່ອເບິ່ງລາຍລະອຽດ

##### ປະຫວັດການຮັບເຂົ້າ/ເບີກອອກ

**ຂັ້ນຕອນທີ 1:** ເລືອກຊ່ວງວັນທີ

**ຂັ້ນຕອນທີ 2:** ເລືອກປະເພດ
- **ທັງໝົດ**: ສະແດງທັງ IN ແລະ OUT
- **IN**: ສະເພາະຮັບເຂົ້າ
- **OUT**: ສະເພາະເບີກອອກ

**ຂັ້ນຕອນທີ 3:** ກົດ **🔍 ຄົ້ນຫາ**

### 5️⃣ 🔧 MAINTENANCE (ບຳລຸງຮັກສາເຄື່ອງຈັກ)

#### ຈຸດປະສົງ
ຈັດການການບຳລຸງຮັກສາເຄື່ອງຈັກ ແລະ ອຸປະກອນ

#### ວິທີໃຊ້ງານ

##### ການບັນທຶກການບຳລຸງຮັກສາ

**ຂັ້ນຕອນທີ 1:** ກົດປຸ່ມ **➕ ເພີ່ມການບຳລຸງ**

**ຂັ້ນຕອນທີ 2:** ປ້ອນຂໍ້ມູນ
- **ເລກທີ Log**: ລະບົບສ້າງອັດຕະໂນມັດ
- **ວັນທີ**: ວັນທີບຳລຸງຮັກສາ
- **ເຄື່ອງຈັກ**: ເລືອກຊື່ເຄື່ອງຈັກ
- **ປະເພດ**: ເລືອກປະເພດການບຳລຸງ
  - `Preventive`: ບຳລຸງຕາມແຜນ
  - `Corrective`: ບຳລຸງແກ້ໄຂບັນຫາ
- **ບັນຫາທີ່ພົບ**: ອະທິບາຍບັນຫາ
- **ການແກ້ໄຂ**: ວິທີການແກ້ໄຂ
- **ວັນບຳລຸງຄັ້ງຕໍ່ໄປ**: ກຳນົດວັນນັດໝາຍ

**ຂັ້ນຕອນທີ 3:** ກົດ **💾 ບັນທຶກ**

##### ການຕິດຕາມການບຳລຸງຮັກສາ

**ຂັ້ນຕອນທີ 1:** ເລືອກຊ່ວງວັນທີ

**ຂັ້ນຕອນທີ 2:** ເລືອກເຄື່ອງຈັກ (ຖ້າຕ້ອງການ)

**ຂັ້ນຕອນທີ 3:** ກົດ **🔍 ຄົ້ນຫາ**

**ຂັ້ນຕອນທີ 4:** ຊົມປະຫວັດການບຳລຸງ

### 6️⃣ ⚙️ SETUP (ຕັ້ງຄ່າລະບົບ) - Admin ເທົ່ານັ້ນ

#### ຈຸດປະສົງ
ຈັດການຕັ້ງຄ່າລະບົບພື້ນຖານ

#### ຫົວຂໍ້ການຕັ້ງຄ່າ

##### 1. ຈັດການລາຍການກວດ (Test Master)

**ການເພີ່ມລາຍການກວດໃໝ່:**
1. ກົດ **➕ ເພີ່ມລາຍການ**
2. ປ້ອນຂໍ້ມູນ:
   - **ຊື່ລາຍການ**: ຊື່ການກວດ
   - **ໝວດໝູ່**: ເລືອກ Category
   - **ລາຄາ**: ລາຄາການກວດ (₭)
3. ກົດ **💾 ບັນທຶກ**

**ການແກ້ໄຂ/ລຶບ:**
- ກົດ **✏️** ເພື່ອແກ້ໄຂ
- ກົດ **🗑️** ເພື່ອລຶບ

**ການນຳເຂົ້າຈາກ CSV:**
1. ກຽມໄຟລ໌ CSV ຕາມຮູບແບບ
2. ກົດ **📂 Import CSV**
3. ເລືອກໄຟລ໌
4. ກົດ **Upload**

##### 2. ຈັດການ Package (Test Packages)

**ການສ້າງ Package ໃໝ່:**
1. ກົດ **➕ ສ້າງ Package**
2. ປ້ອນຂໍ້ມູນ:
   - **ຊື່ Package**: ຊື່ຊຸດກວດ
   - **ຄຳອະທິບາຍ**: ລາຍລະອຽດ Package
   - **ລາຄາ**: ລາຄາລວມ
3. ເລືອກລາຍການກວດໃນ Package
4. ກົດ **💾 ບັນທຶກ**

##### 3. ຕັ້ງຄ່າຜົນກວດ (Test Parameters)

**ການຕັ້ງຄ່າຄ່າອ້າງອີງ:**
1. ກົດ **➕ ເພີ່ມ Parameter**
2. ປ້ອນຂໍ້ມູນ:
   - **ລາຍການກວດ**: ເລືອກ Test
   - **ຊື່ Parameter**: ຊື່ຄ່າທີ່ຕ້ອງການ
   - **ປະເພດ Input**: number/text/select
   - **ຫົວໜ່ວຍ**: ຫົວໜ່ວຍວັດແທກ
   - **ຄ່າປົກກະຕິ**: Min - Max
3. ກົດ **💾 ບັນທຶກ**

##### 4. ກຳນົດການໃຊ້ນ້ຳຢາ (Reagent Mapping)

**ການຕັ້ງຄ່າ:**
1. ກົດ **➕ ເພີ່ມ Mapping**
2. ປ້ອນຂໍ້ມູນ:
   - **ລາຍການກວດ**: ເລືອກ Test
   - **ນ້ຳຢາ**: ເລືອກ Reagent
   - **ຈຳນວນໃຊ້**: ຈຳນວນທີ່ໃຊ້ຕໍ່ການກວດ
3. ກົດ **💾 ບັນທຶກ**

##### 5. ຈັດການ Dropdown Options

**ຕົວເລືອກທີ່ສາມາດຕັ້ງຄ່າໄດ້:**
- **Visit Type**: ປະເພດການມາຮັບບໍລິການ
- **Insite**: ສະຖານທີ່ມາຮັບບໍລິການ
- **Doctor**: ລາຍຊື່ແພດ
- **Department**: ລາຍຊື່ພະແນກ
- **Sender**: ຜູ້ສົ່ງຕົວ
- **Lab Destination**: ຫ້ອງທົດລວດປາຍທາງ

**ການເພີ່ມຕົວເລືອກ:**
1. ພິມຂໍ້ມູນໃນຊ່ອງປ້ອນ
2. ກົດ **➕ ເພີ່ມ**
3. ຕົວເລືອກຈະຖືກເພີ່ມເຂົ້າ Dropdown

## ❓ ຄຳຖາມທີ່ພົບເລື້ອຍ (FAQ)

### Q1: ລືມ Password ຕ້ອງເຮັດແນວໃດ?
**A:** ຕິດຕໍ່ Admin ເພື່ອຣີເຊັດລະຫັດ

### Q2: ສາມາດແກ້ໄຂໃບສັ່ງກວດທີ່ບັນທຶກແລ້ວໄດ້ບໍ່?
**A:** ໄດ້, ໄປທີ່ ປະຫວັດສັ່ງກວດ → ຄົ້ນຫາ → ແກ້ໄຂ

### Q3: ວິທີການດຶງຂໍ້ມູນຄົນເຈັບອັດຕະໂນມັດ?
**A:** ປ້ອນ Patient ID ຢ່າງໜ້ອຍ 2 ຕົວອັກສອນ, ລະບົບຈະຄົ້ນຫາອັດຕະໂນມັດ

### Q4: FIFO ໝາຍຄວາມວ່າແນວໃດ?
**A:** First-In-First-Out = ນ້ຳຢາທີ່ເຂົ້າກ່ອນ ຈະຖືກໃຊ້ກ່ອນ

### Q5: ວິທີການ Export ຂໍ້ມູນເປັນ Excel?
**A:** ກົດປຸ່ມ 📊 Export Excel ໃນໜ້າທີ່ຕ້ອງການ

### Q6: Dashboard ບໍ່ສະແດງຂໍ້ມູນ?
**A:** 
1. ກວດສອບຊ່ວງວັນທີ
2. ກວດສອບຕົວກັ່ນຕອງ
3. ກົດປຸ່ມ ໂຫຼດຂໍ້ມູນໃໝ່

### Q7: ຈະຮູ້ໄດ້ແນວໃດວ່ານ້ຳຢາໃກ້ໝົດອາຍຸ?
**A:** ລະບົບຈະແຈ້ງເຕືອນດ້ວຍ:
- Badge ສີແດງດ້ານເທິງ
- ສະຖານະ Expiring Soon (ສີເຫຼືອງ)
- ແຈ້ງເຕືອນເມື່ອເປີດໜ້າ Inventory

### Q8: ສາມາດໃຊ້ງານໃນມືຖືໄດ້ບໍ່?
**A:** ໄດ້, ລະບົບຮອງຮັບ Responsive Design

### Q9: ວິທີການສຳຮອງຂໍ້ມູນ?
**A:** ຂໍ້ມູນຖືກບັນທຶກໃນ Supabase ອັດຕະໂນມັດ, ສາມາດ Export ເປັນ Excel ໄດ້

### Q10: ມີຈຳນວນຜູ້ໃຊ້ງານສູງສຸດເທົ່າໃດ?
**A:** ບໍ່ຈຳກັດ, ສາມາດສ້າງຜູ້ໃຊ້ງານໄດ້ຕາມຕ້ອງການ
